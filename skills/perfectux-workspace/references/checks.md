# Workspace Checks

Use before final response when PerfectUX workspace context or MCP tools were used.

- Correct project or artifact selected.
- Relevant resources were read before proposing writes.
- User-facing summary hides raw MCP JSON.
- Any write used schema-driven tools or the appropriate workspace action.
- Approval state is clear: proposed, pending approval, applied, failed, or no-op.
- Permission or validation failures include a specific recovery action.
- Legacy task fields and `modify_task_proposal` were not used.
- DESIGN.md or Foundation JSON was preferred for implementation handoff context when relevant.
