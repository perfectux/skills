# Project Context

Use the smallest PerfectUX artifact that answers the task.

## Resource Routing

- Project list: choose or confirm the target project.
- Project metadata: understand product name, status, and available artifacts.
- Project tasks: read canonical brand, PRD, design, logo, and related task outputs.
- Single task artifact: inspect or modify a specific canonical task.
- Launch package: work on launch asset workspace data.
- Marketing campaigns: work on marketing campaign and channel assets.
- Docs: work on document workspace assets.
- DESIGN.md export: provide implementation-ready design context.
- Foundation JSON: provide structured project context for downstream tools.

## Context Rules

- Do not infer missing project facts when MCP can read them.
- Preserve PerfectUX product terminology from resources.
- State assumptions when MCP data is incomplete or unavailable.
- Do not treat generated content as verified unless its source or artifact state supports that claim.
